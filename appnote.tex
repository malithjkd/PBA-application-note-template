\section{Background and Theory}

The PBA GS-Series gantry operates in force control mode by replacing
the position error signal with a force error derived from an analogue
load cell.  The control relationship is:

\begin{equation}
  F_{\text{applied}} = k_f \cdot I_{\text{cmd}} - F_{\text{preload}}
\end{equation}

where $k_f$ is the motor force constant [\si{\newton\per\ampere}] and
$F_{\text{preload}}$ is the Z-axis gravitational preload [\si{\newton}].

\section{Prerequisites and Safety Notices}

\begin{dangerbox}
  The Z-axis motor exerts forces up to \SI{500}{\newton}. Never place hands
  under the Z-axis head while servos are enabled. Use the Emergency Stop
  (E-Stop) when adjusting hardware with the machine energised.
\end{dangerbox}

\bigskip

\begin{warningbox}
  Force control mode removes position-based end-of-travel protection.
  An incorrectly tuned loop can drive the axis into hard contact at high
  current. Set a conservative force limit before enabling the loop for
  the first time.
\end{warningbox}

\bigskip

\begin{cautionbox}
  The load cell cable shield must be grounded at the signal conditioner
  end only.  Grounding at both ends causes a \SI{50}{\hertz} ground loop
  and introduces noise into the force signal.
\end{cautionbox}

\bigskip

\begin{notebox}
  Parameter changes are non-persistent by default.  Use
  \menu{File > Save Configuration} in the ACS MMI after confirming
  stable operation.
\end{notebox}

\section{Procedure}

\subsection{Phase 1 — Enable Servo and Zero Sensor}

\begin{warningbox}
  Ensure no tooling or workpiece is under the Z-axis head before enabling
  the servo.
\end{warningbox}

\begin{steplist}
  \step Enable the Z-axis servo from the ACS MMI terminal:
    \begin{acscode}[Enable Z-axis servo]
\begin{lstlisting}
; Enable servo on Axis 0
ENABLE 0
WAIT 100        ; Wait 100 ms to stabilise
PRINT "Servo state: ", MFLAGS(0)
\end{lstlisting}
    \end{acscode}
  \outcome{The axis indicator in the ACS MMI turns green.
           \acsparam{MFLAGS} bit~0 is set.}

  \step Zero the force sensor (no load applied):
    \begin{acscode}[Zero the force sensor offset]
\begin{lstlisting}
GLOBAL REAL force_offset
force_offset = AIN(0, 1)
PRINT "Offset set to: ", force_offset, " V"
\end{lstlisting}
    \end{acscode}
  \outcome{\code{force\_offset} is stored.  Value should be within
           $\pm$\SI{0.05}{\volt}.}

  \step Set PID gains and current limit:
    \begin{acscode}[Set force loop parameters]
\begin{lstlisting}
KP(0) = 2.5
KI(0) = 0.08
KD(0) = 0.001
DCURR(0) = 1.5
PRINT "Gains set. KP=", KP(0)
\end{lstlisting}
    \end{acscode}
\end{steplist}

\subsection{Phase 2 — Python Force Logging}

Run the logger from the host PC while the Z-axis makes contact:

\begin{pythoncode}[force\_logger.py — launch from terminal]
\begin{lstlisting}
# Navigate to the project folder
cd C:\PBA\ForceControl\scripts

# Log for 30 seconds at 100 Hz
python force_logger.py --ip 10.0.0.100 --duration 30

# Quick check — print stats from the log
import pandas as pd
df = pd.read_csv("force_log.csv")
print(f"Max force: {df['force_N'].max():.2f} N")
print(f"Noise RMS: {df['force_N'].std():.4f} N")
\end{lstlisting}
\end{pythoncode}
